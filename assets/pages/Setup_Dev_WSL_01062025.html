Complete guide to setting up a modern development environment using Windows Subsystem for Linux

Windows Subsystem for Linux (WSL) has revolutionized development on Windows machines. It provides a seamless Linux experience without the overhead of traditional virtual machines. In this comprehensive guide, we'll walk through setting up a complete development environment that rivals any native Linux setup.

Why WSL for Development?

WSL bridges the gap between Windows and Linux, offering developers the best of both worlds. You get access to powerful Windows applications while enjoying the flexibility and tools of a Linux environment. This is particularly valuable for web development, where many deployment targets are Linux-based.

"WSL has completely transformed my development workflow. I can now use Linux tools natively while still having access to my favorite Windows applications." - A satisfied developer

Prerequisites

Before we begin, ensure you have:

  • Windows 10 version 2004 and higher (Build 19041 and higher) or Windows 11
  • Administrator privileges on your machine
  • At least 4GB of free disk space
  • A stable internet connection for downloads

Step 1: Enable WSL

First, we need to enable the Windows Subsystem for Linux feature. Open PowerShell as Administrator and run: 

dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart

Enable Virtual Machine Platform

For WSL 2, we also need to enable the Virtual Machine Platform:

dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

Step 2: Install Ubuntu

Ubuntu is one of the most popular Linux distributions and works excellently with WSL. You can install it directly from the Microsoft Store or use the command line: 

wsl --install -d Ubuntu

After installation, launch Ubuntu from the Start menu. You'll be prompted to create a username and password for your Linux environment.

Step 3: Update Your System

Once Ubuntu is running, update the package lists and upgrade existing packages:

sudo apt update && sudo apt upgrade -y

Step 4: Install Essential Development Tools

Git Configuration

Git is essential for version control. Configure it with your details:

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Node.js and npm

For JavaScript development, install Node.js using Node Version Manager (nvm):

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install --lts
nvm use --lts

Python Development

Install Python and pip for Python development:

sudo apt install python3 python3-pip python3-venv -y

Step 5: Configure Your Shell

Enhance your terminal experience with Zsh and Oh My Zsh for better productivity: 

sudo apt install zsh -y
sh -c "$(curl -fsSL https://raw.github.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

Step 6: Install VS Code Integration

Visual Studio Code has excellent WSL integration. Install the Remote-WSL extension to edit files directly in your WSL environment from VS Code running on Windows.

Opening Projects in VS Code

From your WSL terminal, you can open any directory in VS Code:

code .

Best Practices and Tips

  • Store your projects in the Linux filesystem (/home/username/) for better performance
  • Use Windows Terminal for a better terminal experience
  • Set up SSH keys for seamless Git operations
  • Consider using Docker Desktop with WSL 2 backend for containerized development
  • Regularly backup your WSL environment using wsl --export

Troubleshooting Common Issues

WSL 2 Not Starting

If WSL 2 fails to start, ensure virtualization is enabled in your BIOS and that Hyper-V is properly configured.

Slow File Performance

Always work within the Linux filesystem for optimal performance. Avoid accessing Windows files from WSL when possible.

Conclusion

Setting up a development environment in WSL provides a powerful, flexible platform for modern software development. With this setup, you'll have access to the vast ecosystem of Linux tools while maintaining the familiarity of Windows.

The combination of WSL, VS Code, and modern development tools creates an environment that's both productive and enjoyable to work in. Whether you're developing web applications, working with containers, or building cloud-native solutions, this setup will serve you well.
